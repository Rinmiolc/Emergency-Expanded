# 生理与病理系统架构设计规范 (中文版)

本篇文档定义了 **Emergency Expanded** 模组中生理与病理模拟系统的架构原则与设计规范。旨在维持高拟真医学模拟的同时，彻底杜绝“恶性循环”（无限反馈环/循环依赖）问题。

---

## 1. 核心痛点：恶性反馈循环

在医学模拟中，常见的误区是将机体功能能力扣减与病理状态积累直接耦合。例如：
$$\text{灌注不足} \longrightarrow \text{酸中毒加深} \longrightarrow \text{立即降低泵血（setMax）} \longrightarrow \text{更低的灌注}$$
这会导致小人的体征在短时间内瞬间雪崩崩塌，阻断了抢救拉锯的可能。即使后期通过输血补充了容量，已被扣死的泵血效能导致机体无法自我清偿代谢负债，造成逻辑死锁。

---

## 2. 分层单向生理模型

为了解决这一问题，生理病理系统被严格划分为四个递进的层次。状态更新以**单向向下**为主流向；逆向反馈被严格限制为异步、延迟的器质性阈值。

```
+-----------------------------------------------------------+
| 第一层：原发物理损伤 (Primary Insults)                     |
| - 外伤、活动性大出血、气胸、烧伤、心肌梗死/室颤             |
+-----------------------------+-----------------------------+
                              | (直接物理影响)
                              v
+-----------------------------------------------------------+
| 第二层：基础生理泵效 (Base Capacities)                     |
| - 泵血效能、呼吸效能、血容量                                |
+-----------------------------+-----------------------------+
                              | (功能缺口转化为负债)
                              v
+-----------------------------------------------------------+
| 第三层：病理生理与生理负债 (Pathophysiology & Debt)         |
| - 休克严重度、代谢性酸中毒（pH）、全身炎症（SIRS）         |
+-----------------------------+-----------------------------+
                              | (持续负债转化为实质器质损伤)
                              v
+-----------------------------------------------------------+
| 第四层：终末器官衰竭与死亡 (Terminal Failure)              |
| - 多器官功能障碍综合征（MODS）、脑缺氧、死亡倒计时          |
+-----------------------------------------------------------+
                              |
                              | (异步能力上限限制反馈)
                              + - - - - - - - - - - - - - - - > [反馈至第二层]
```

### 第一层：原发物理损伤
*   **定义**：由外部伤害或事件直接触发的物理损伤，作为生理系统的源头输入。
*   **示例**：枪伤、活动性大出血、骨折、烧伤、张力性气胸、冠状动脉阻塞（心肌梗死/室颤）。

### 第二层：基础生理泵效
*   **定义**：仅基于器官物理完整度与血容量计算出的基础生理能力。
*   **示例**：泵血效能、呼吸效能、血容量。
*   **准则**：这些能力**仅**受第一层原发状态（零件物理缺失、缺损、大出血）的影响，**严禁**受第三层病理状态（如酸中毒、休克）的直接限制（如 setMax 或直接乘数扣减）。

### 第三层：病理生理与生理负债
*   **定义**：累积的“生理负债”或“代谢债”。
*   **示例**：休克严重度、代谢性酸中毒（pH）、全身炎症（SIRS）。
*   **准则**：第二层能力的缺乏（如低泵血、低血容量）会在第三层随着时间转化为生理负债；第二层能力恢复时，负债以恒定速度清偿。**本层绝对不允许直接施加 `setMax` 扣减第二层的泵血与呼吸。**

### 第四层：终末器官衰竭与死亡
*   **定义**：因第三层生理负债维持时间过长，导致脏器发生的实质性物理损伤与崩溃。
*   **示例**：多器官功能障碍综合征（MODS）、脑缺氧、生物学死亡倒计时。
*   **准则**：**只有第四层的状态允许向第二层进行能力上限限制反馈**（如 MODS 限制泵血最大为 20%）。由于负债转化为实质脏器衰竭需要相当长时间，这确保了玩家拥有充足的“黄金抢救期”进行补液、用药和换心手术。

## 3. 数学模型：生理负债

为了防止机体瞬间崩盘，使用**累积率（积分）**代替即时能力减成。

### 旧版死锁逻辑（弃用）：
$$\text{Acidosis.Severity} \ge 0.5 \implies \text{Pumping.setMax} = 0.5$$

### 单向负债逻辑（规范）：
1. **酸中毒变化率**：
   $$\Delta\text{Acidosis} = (\text{灌注缺口} - \text{代偿阈值}) \times K$$
   其中 $\text{灌注缺口} = 1.0f - \text{Pumping}$。
2. **酸中毒恢复**：
   当 $\text{Pumping} \ge 0.8f$ 且无休克压力时，酸中毒以恒定速度下降：
   $$\Delta\text{Acidosis} = -\text{DecayRatePerDay} / 1000f$$
3. **器官衰竭（MODS）累积**：
   $$\Delta\text{MODS.Severity} = (\text{Shock.Severity} \times 0.5 + \text{Acidosis.Severity} \times 0.5) \times \text{DamageRate}$$
4. **器官衰竭限制**：
   仅 `MODS.Severity` 允许应用最大能力限制（如晚期限制 `setMax` 0.2）。

## 4. C# 时序执行规范

为了消除时序竞争与悬挂更新，所有生理状态必须在 `CompEE_PawnGizmos.CompTick` 中每 60 刻度由**统一的生理调度器**处理。执行顺序必须严格遵循：

```csharp
public static void RunUnifiedPhysiologyTick(Pawn pawn)
{
    // 1. 物理数据收集 (Data Collection)
    // 仅基于心脏零件、血容量等物理状态计算泵血/呼吸效能。严禁读取休克或酸中毒乘数！

    // 2. 生理负债更新 (Debt Accumulation)
    // 依据灌注与炎性负载计算压力，并更新休克、酸中毒和 SIRS 的进度。

    // 3. 器质性损伤判定 (Organ Damage)
    // 依据持续休克、酸中毒或无灌注时间，更新 MODS 和脑缺氧进度。

    // 4. 终末能力施加 (Capacity Constraints)
    // 应用第四层终末器官衰竭状态对第二层施加的最终 getSetMax 能力限制。
}
```

遵循这套开发规范，将确保 Emergency Expanded 在实现极致真实感的同时，提供强健、无死锁、给玩家操作提供正向反馈的战地医疗抢救玩法。
